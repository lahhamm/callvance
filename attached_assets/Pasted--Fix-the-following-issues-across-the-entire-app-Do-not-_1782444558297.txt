4. Show DB bookings in client portal

The client portal Appointments tab currently only shows the Cal.com embed. The booking data is already being fetched from GET /api/client/:token/bookings but is only used for the stats counter. Add a section above or below the Cal.com embed that lists the confirmed bookings from the database. Each row should show: contact name, phone number, date and time (in the client's timezone), and notes. If there are no upcoming bookings, show an empty state message. Keep the Cal.com embed — just add the DB bookings list alongside it.
5. Bulk call button in UI

The POST /api/admin/clients/:id/calls/bulk endpoint exists and works but is never called from the frontend. Add a "Call All" button to the Contacts tab on the client detail page. It should: select all contacts, confirm with a dialog ("This will call X contacts. Are you sure?"), then POST to the bulk endpoint with all contact IDs. Show a toast when done with how many succeeded and how many failed. Disable the button while calls are in progress.
